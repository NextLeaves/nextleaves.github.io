NET Application Frame

1.1 选择技术

  • 针对开发不同的应用程序（windows、web），服务的不同技术信息，选择合适的开发工具

1.2 回顾.NET历史

  • 知道新技术还不够，应该了解新技术在旧技术上面的迭代和方式和运作原理
  • .NET 1.0 – CLR 1.0 – C# 1.0 – Visual Studio 2002

1.2.1 C# 1.0

  • 开发java的sun公司，不同意微软修改java用于.net framework开发
  • 设计者Anders Hejlskerg（安德斯·海尔斯伯格）
  • 借鉴于java和c++的特点：
    • if语句中的boolean
    • struct和class定义结构类型和类类型（java只有引用类型，即class）
    • 允许使用虚拟方法和非虚拟方法（java都是虚拟方法）
  • 编程语言都有自己的运行库：
    • C++运行库
    • VB运行库为VBRun
    • Java运行库为JVM
  • CLR是每种.NET编程语言都要使用的运行库
    • JSScript.NET - JS编译器
    • Visual Studio.NET - Visual Basic
    • Managed C++ - C++/CLR
    • C# - C#编译器
  • .NET编译器生成中间语言IL
    • 使用程序ildasm.exe可以反编译，获取中间语言
    • CLR包含一个即时编译器(JIT),程序运行的时候，JIT会把IL编译为本地代码；IL中间语言，又称托管代码
    • CLR其他为：
      • GC，垃圾回收机制
      • Debug，调试器扩展
      • Thread，线程实用工具
  • .NET Framework第一版特点：
    • 名称空间解决类名冲突
    • Windows Forms（创建窗口应用程序）
    • ASP.NET Web Forms（创建Web应用程序）
    • ASP.NET Web Services（应用程序与Web服务通信）
    • .NET Remoting（.NET应用程序之间更快的通信）
    • Enterprise Forms（应用程序服务器上的COM+组件）

1.2.2 C#2和.NET2

  • 泛型的创建（System.Collections.Generic;）
  • 1.0中的就集合类很少使用

1.2.3 .NET3.0时代

  • 创建Windos桌面应用程序（WPF）
  • Windows Forms与WPF区别：
    • 前者基于像素，后者基于DirectX
    • 后者使用矢量图，无缩放调整
    • 后者允许完全自定义外观
  • System.Windos；下全部内容为WPF，除System.Windos.From;
  • 用户界面使用XML语法设计XAML（XML For Applicaiton Markup Language）
  • ASP.NET Web Services和.NET Remoting用于web通信
  • Message Queuing用于通信的另一个选择
  • 企业应用程序需要用几种通信API，学习成本高
    • WCF：
      • 把其他API集中体现到一个API中
      • 但为支持WCF的所有功能，需要配置WCF
  • Windos WF(WrokFlow Foundation)和System.Workflow
    • 不是为了应用程序创建自定义的工作流程序，而是把工作流引擎作为应用程序的一部分
  • 8000个类增加到12000个类

1.2.4 C#3.0和.NET3.5时代 -LINQ

  • C#定义的查询语法，允许使用相同的语法过滤和排序对象列表、XML文件、数据库
  • 语言增强，使用更少的代码，达到相同的功能
  • LINQ TO SQL为.NET3.5的一部分，但EntityFramework更强大
    • 两者都提供从层次结构到数据库映射和LINQ提供程序
    • 前者简单，后者强大
  • System.AddIn
    • 提供插件模型；外部运行插件的强大功能；使用较为复杂

1.2.5 C#4和.NET4.0 - dynamic 和TPL

  • 动态集成脚本，更容易使用COM集成
  • C#语法扩展dynamic关键字、命名参数和可选参数以及泛型增强的协变和逆变
  • 任务并行库（TPL，Task Parallel library）
  • Task类和Parallel类抽象出线程，易于编程
  • 旧引擎workflow未实现应有功能，在此发布全新的Workflow Engine(System.Activity)
  • 运行库的升级，从版本2调到版本4
  • 发布VS2010，同时发布一款创建Web应用程序的新技术-ASP.NET MVC2.0
    • 与ASP.NET Web Froms不同的是，需要使用HTML和JS

1.2.6 C#5和异步编程

  • async和await，大大简化异步编程
    • 不阻塞UI线程，提高响应
    • 收拾的反应速度
  • Windos Store应用程序（Modern应用程序，Metro应用程序，通用Windos应用程序，Windos应用程序）
    • Windos8中的Windos Store开发新的API（实际为底层映射代码）
    • 称为Windos运行库
  • System.AddIn过于复杂
  • 独立于平台的通信协议新版本：ASP.NET Web API
    • WCF提供有状态，无状态服务；以及不同的网络协议
    • ASP.NET Web API更简单，基于RESTCRepresentational State Transferl软件架构风格

1.2.7 C#6和.NET Core

  • 许多小而是用的语言增强；通过新的编译器引起Roslyn来实现
  • Microsoft Silverlight是Web浏览器插件，支持动态内容
    • 代码WPF/E即WPF Everywhere
    • 第一个版本只支持JS
  • .NET框架的巨大，导致更新周期长；现今开发快节奏的时代，也需要大的改进
    • 创建Fremework新版本，.NETCore
    • .NET Core
      • 模块化的NuGet包，以及每个应用程序的运行库是开源的
      • 不仅用于Windos桌面版，许多不同Windos设备
      • 以及LInus和OS X中
    • ASP.NET Core 1.0 为创建Web应用程序
    • 不是所有应用程序都易于改变为.NET Core方式
      • .NET Framework虽然更新周期更长，但是依然会更新下去

1.2.8 选择技术，继续前行

  • 旧技术 - 新技术：
    • Windos Forms - WPF
    • Web Forms - MVC6/.NET Core
    • LINQ to SQL - Entity Framework
    • System.AddIn - System.Activity.Managed;
  • 只为改变现有应用程序而是用新技术是没有意义的
    • 维护代码周期长
    • 开发周期长
    • 以上问题可以适当考虑换新的语言方式
